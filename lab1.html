¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red de computadoras conectadas entre sí alrededor de todo el mundo, por otra parte, la World Wide Web es una colección de páginas que se asienta sobre esa red de computadoras.

¿Cuáles son las partes de un URL?

Protocolo, dominio, subdominio, ruta, parámetro y etiqueta.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: El método GET solicita datos de un recurso específico.

HEAD: El método HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.

POST: El método POST se utiliza para enviar una entidad a un recurso en específico.

PUT: El modo PUT reemplaza todos los datos actuales del recurso de destino con la carga del dato a petición.

PATCH: El método PATCH es utilizado para aplicar modificaciones parciales a un recurso.

DELETE: El método DELETE borra un recurso en específico.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Los métodos GET y POST: el método GET envía los datos usando la URL y el método POST los envía de forma que no podemos verlos.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Las peticiones al servidor suelen realizarse utilizando el método de petición GET, en el que el recurso se solicita a través de la URL al servidor web.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Significa que la solicitud ha tenido éxito.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, ya que un error 404 indica que la página que hemos solicitado no se encuentra en el servidor.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

No, ya que indica que el servidor encontró una condición inesperada que le impide completar la petición.
