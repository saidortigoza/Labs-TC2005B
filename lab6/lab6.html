¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Porque en JavaScript al validar los datos de los inputs ingresados podemos notificar al instante al usuario si cometió algún error.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Utilizando algunas palabras reservadas, lo que significa obtener los valores de las variables enviadas a través de métodos como POST y con ayuda del operador $.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Es una buena práctica porque se le notifica al instante al usuario si existe algún error. Sin embargo se deben utilizar algoritmos de encriptación para asegurar que no cualquiera pueda acceder a la información de las variables.
